# LinkedIn Post Draft: Building My Personal Website with Claude

---

For most of my career, personal brand was an afterthought.

I was focused on Secureworks. Then PhishLabs. Then Fortra. The work was the priority. My own presence online? Something I'd get to eventually.

Eventually arrived when I started looking for my next leadership role.

---

I had two goals going in. One was obvious: a stronger platform for my own brand, somewhere to publish my thinking that wasn't just LinkedIn. The other was less obvious: I wanted hands-on AI experience. Not workshops or webinars. Actual experience using AI tools to build something.

Building a personal website checked both boxes. So I started.

---

I've built websites before, but it's been a while. I was rusty. The stack I landed on (Jekyll, GitHub Pages, eventually Netlify) wasn't something I'd have set up quickly on my own. That's where Claude came in.

I started with Claude Chat, which walked me through every decision: which hosting platform, how to structure the Jekyll project, how to migrate a single HTML file into a proper site with an About page and a blog. I still had to make judgment calls. I still hit errors I had to work through. But the time to get from "I want a site" to "I have a live site" was a fraction of what it would have been otherwise.

The bigger unlock was Cowork, Claude's desktop app.

In Cowork, Claude works directly in my project folder instead of passing files back and forth in a chat window. That changed everything. I automated essentially every manual task in the pipeline (building, editing, debugging, security checks) except pushing code live. What used to take me an afternoon of hunting through documentation took a conversation.

Bug hunting and basic security checks especially. I didn't run tools myself or go digging. I described what I wanted to verify, and Claude did the work.

---

**A few things I'd tell someone starting the same project:**

**You probably don't need a CMS.** I spent real time evaluating CloudCannon, Decap CMS, and Prose.io before realizing I didn't need any of them. With Claude handling the coding and file management, there was no gap to fill. Dropping the CMS simplified the entire stack. That's a decision I wouldn't have made without seeing how the workflow actually played out.

**Iteration speed is the real value.** I spent a few hours a week on this, in between other projects and commitments. The site evolved faster than I expected because every change (design, layout, content structure) took minutes instead of hours. I could try something, see it wasn't right, and try something else without losing momentum.

**AI helps with structure, not substance.** My writing is my writing. I used Claude for light editing and to break content into formats that work better on the web. It didn't generate my thinking or my voice. What it did was get out of the way and let me focus on the content instead of the plumbing.

---

The site isn't finished. Sites never are. But it exists, it reflects how I actually think about my work, and I built it in less time than I expected while learning something genuinely useful in the process.

Not a bad trade for a few hours a week.
